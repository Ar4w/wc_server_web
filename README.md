# 💧 WC Server v3 (ESP32 Water Meter Reader)

Автоматизированная IoT-система на базе микроконтроллера **ESP32** для снятия показаний счетчиков воды (через импульсные герконы) и их автоматической передачи в личный кабинет **МОСОБЛЕИРЦ** (Московская область).

Проект представляет собой полностью переработанную версию с отказом от Telegram-бота в пользу встроенного **Web-интерфейса** и **Captive Portal** для бесшовной первоначальной настройки.

---

## 🚀 Ключевые возможности

- 🌐 **Captive Portal**: Автоматическая точка доступа для настройки Wi-Fi и параметров ЕИРЦ без необходимости перепрошивки устройства.
- 📊 **Веб-интерфейс**: Удобная панель управления для просмотра показаний, ручной отправки, экспорта/импорта конфигурации и мониторинга состояния системы.
- 🛡️ **Защита от сбоев**: 
  - Блокировка отправки при аномальном расходе (>30 м³/мес) для предотвращения катастрофических ошибок при залипании геркона.
  - Режим "Dry Run" (тестовая отправка) для безопасной отладки.
- 💾 **Надежное хранение**: Использование NVS (Non-Volatile Storage) с встроенным wear-leveling и CRC-верификацией. Данные не теряются при внезапном отключении питания.
- ⚡ **Оптимизация сети**: Специальные настройки драйвера Wi-Fi (`WIFI_PS_NONE`, фиксированная мощность) для стабильной работы в загруженных домашних сетях.
- ⏱️ **Асинхронная архитектура**: HTTP-запросы к ЕИРЦ выполняются в фоновых задачах FreeRTOS, что предотвращает срабатывание Watchdog Timer (WDT) и блокировку веб-сервера.

## 🛠️ Аппаратные требования

| Компонент | Описание |
|-----------|----------|
| **Контроллер** | ESP32 Dev Module (протестировано на ESP32-WROOM-DA) |
| **Датчики** | Герконовые датчики или импульсные выходы счетчиков (замыкание на GND) |
| **Подключение** | Горячая вода: **GPIO 23** <br> Холодная вода: **GPIO 18** <br> Земля: **GND** |
| **Подтяжка** | Программная (`INPUT_PULLUP`), внешние резисторы не требуются |
| **Питание** | Стабилизированные 5V (через USB) или 3.3V напрямую |

## 💻 Программное окружение

Для сборки проекта требуется **Arduino IDE 2.x** (или PlatformIO) со следующими настройками:

1. **Плата**: `ESP32 Dev Module` (Core версии 3.x)
2. **Partition Scheme**: `Default 4MB with spiffs` (или `Minimal SPIFFS`)
3. **Необходимые библиотеки** (установить через Менеджер библиотек):
   - `ESPAsyncWebServer` (автор: `ESP32Async`)
   - `AsyncTCP` (автор: `ESP32Async`)
   - `ArduinoJson` (версия `6.21.x` или выше)

## 📦 Установка и прошивка

### 1. Подготовка файловой системы (LittleFS)
Веб-интерфейс хранится в файловой системе ESP32. Перед прошивкой кода необходимо загрузить файлы из папки `data/`.
1. Установите плагин **ESP32 Sketch Data Upload** для Arduino IDE. https://github.com/earlephilhower/arduino-littlefs-upload
2. Убедитесь, что папка `data` (с файлами `index.html`, `style.css`, `script.js`) находится в корне проекта.
3. В Arduino IDE выберите: `Инструменты` → `ESP32 Sketch Data Upload`.
4. Дождитесь сообщения об успешной загрузке.

### 2. Прошивка микроконтроллера
1. Откройте файл `wc_server_v3.ino`.
2. Выберите порт и плату.
3. *(Рекомендуется при первом запуске)*: `Инструменты` → `Erase All Flash Before Sketch Upload` → `Enabled`.
4. Нажмите **Загрузить** (Upload).

## ⚙️ Первоначальная настройка (First Run)

1. После прошивки устройство создаст открытую точку доступа Wi-Fi с именем: **`WC_Server_Config`**.
2. Подключитесь к этой сети с телефона или ноутбука.
3. Откройте браузер. Вы будете автоматически перенаправлены на страницу настройки (Captive Portal). Если нет, перейдите по адресу `http://192.168.4.1`.
4. Заполните форму:
   - **Wi-Fi SSID / Пароль**: Данные вашей основной домашней сети.
   - **EIRC Логин / Пароль**: Телефон и пароль от личного кабинета МОСОБЛЕИРЦ.
   - **ID счетчиков**: Уникальные числовые идентификаторы. 
     > 💡 *Совет: Используйте прилагаемый скрипт `get_data_from_eirc.py` на компьютере, чтобы точно узнать ID ваших счетчиков.*
5. Нажмите **💾 Сохранить и перезагрузить**. Устройство сохранит данные в NVS, перезагрузится и подключится к вашей основной сети.

## 🖥️ Работа с веб-интерфейсом

После подключения к основной сети узнайте IP-адрес устройства (через веб-интерфейс роутера или монитор порта) и откройте его в браузере.

- **📊 Текущие показания**: Отображает накопленный объем в десятках литров. Кнопка принудительной отправки.
- **⚙️ Установка показаний**: Ручная коррекция начальных значений (например, если счетчик показывает `00145,765`, введите `14576`).
- **🏢 МОСОБЛЕИРЦ**: Статус авторизации, разрешенное окно подачи (например, 14–19 число) и дата ближайшей поверки.
- **🔧 Настройки**: Изменение параметров, экспорт/импорт конфигурации в JSON, сброс к заводским настройкам.

## 🔒 Безопасность и надежность

1. **Защита от дребезга**: Аппаратный таймер опрашивает пины каждые 100 мс. Импульс регистрируется только при стабильном изменении состояния, что исключает ложные срабатывания.
2. **Лимит потребления**: Перед отправкой система сравнивает новые показания с последними данными от ЕИРЦ. Если разница превышает `30 м³`, отправка блокируется, и в лог выводится предупреждение `LIMIT_EXCEEDED`.
3. **Фоновая синхронизация**: При каждом запуске и синхронизации NTP устройство автоматически запрашивает актуальные показания с сервера ЕИРЦ, чтобы корректно рассчитывать дельту расхода.

## 🐛 Диагностика и устранение неполадок

| Проблема | Возможная причина | Решение |
|----------|-------------------|---------|
| Не открывается Captive Portal | Включен VPN или мобильный интернет на телефоне | Отключите мобильные данные и VPN, вручную введите `192.168.4.1` |
| Ошибка `Limit exceeded` при отправке | Неверно указаны ID счетчиков или устройство не успело получить данные с сервера | Проверьте ID через скрипт Python. Нажмите "Обновить данные ЕИРЦ" и дождитесь логов в Serial Monitor |
| Устройство часто теряет Wi-Fi | Агрессивный режим энергосбережения или Band Steering | В коде уже применены оптимизации (`WiFi.setSleep(false)`). Убедитесь, что роутер использует канал 20 МГц только на 2.4 ГГц |
| Ошибка `Not Found` при открытии IP | Файлы LittleFS не были загружены | Выполните `ESP32 Sketch Data Upload` заново |
